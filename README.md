### 🏥 SmartClinic — Hospital Management System

[![Repo](https://img.shields.io/badge/Repository-181717?style=for-the-badge&logo=github&logoColor=white)](https://github.com/Pruthvi226/SmartClinic_Management_System)
[![Live](https://img.shields.io/badge/Live_Demo-0B0D0E?style=for-the-badge&logo=railway&logoColor=white)](https://smartclinicmanagementsystem-production.up.railway.app)
[![Java](https://img.shields.io/badge/Java_17-ED8B00?style=flat-square&logo=openjdk&logoColor=white)]()
[![Spring](https://img.shields.io/badge/Spring_MVC-6DB33F?style=flat-square&logo=spring&logoColor=white)]()
[![MySQL](https://img.shields.io/badge/MySQL_8-4479A1?style=flat-square&logo=mysql&logoColor=white)]()
[![Docker](https://img.shields.io/badge/Docker-2496ED?style=flat-square&logo=docker&logoColor=white)]()

**Problem Solved:** Managing hospital operations — appointments, pharmacy, billing, patient records — is fragmented and error-prone without a unified system. SmartClinic centralizes all workflows across four roles (Admin, Doctor, Receptionist, Pharmacist) with intelligent slot allocation, automated billing, and enterprise-grade security.

<details>
<summary><b>📋 Key Features & Architecture</b></summary>

**Features:**
- 🔁 **Smart Slot Allocator** — `SlotAllocator.java` schedules appointments using priority queues, respecting doctor leave dates and availability windows
- 👥 **4-Role Workflow Engine** — Admin, Doctor, Receptionist, Pharmacist each with isolated, permission-gated dashboards
- 🧾 **Automated Billing Pipeline** — Invoice generation, tax calculation, discount logic, payment status tracking
- 💊 **Pharmacy Management** — Inventory, expiry alerts, low-stock warnings, prescription-linked dispensing
- 🔒 **Enterprise Security** — BCrypt, CSRF, RBAC with Spring Security, parameterized queries, audit trail
- 🐳 **Production Deployment** — Multi-stage Docker build, Docker Compose with health checks, Railway hosting

**Architecture:**
```
Browser / REST Client
    ↓
DispatcherServlet (Spring MVC)
    ↓
@Controller — request routing & JSP rendering
    ↓
@Service — business logic, @Transactional boundaries
    ↓
@Repository / DAO — Hibernate SessionFactory
    ↓
MySQL 8.0 — 12 tables, stored procedures, audit logs
```

</details>

### 🏙️ CivicFix+ — Smart Civic Issue Resolution Platform

[![Repo](https://img.shields.io/badge/Repository-181717?style=for-the-badge&logo=github&logoColor=white)](https://github.com/Pruthvi226/CivicFix)
[![Java](https://img.shields.io/badge/Java_21-ED8B00?style=flat-square&logo=openjdk&logoColor=white)]()
[![Spring](https://img.shields.io/badge/Spring_MVC-6DB33F?style=flat-square&logo=spring&logoColor=white)]()
[![MySQL](https://img.shields.io/badge/MySQL_8-4479A1?style=flat-square&logo=mysql&logoColor=white)]()

**Problem Solved:** Civic complaints are usually submitted into a black hole — no tracking, no accountability, no resolution SLAs. CivicFix builds transparent, AI-assisted municipal workflows for complaint resolution with intelligent classification, duplicate detection, and performance incentives.

<details>
<summary><b>📋 Key Features & Architecture</b></summary>

**Features:**
- 🤖 **AI Complaint Classification** — NLP keyword engine infers category (road/water/sanitation) and severity automatically
- 📍 **Duplicate Radar** — Haversine formula finds existing complaints within a 100m radius, preventing duplication
- 🏆 **Karma Reward System** — Verified citizens earn points; full ledger with transaction history
- 🔮 **Predictive Maintenance** — ML-based risk scores flag zones for proactive maintenance advisories
- 🕵️ **Whistleblower Mode** — AES-encrypted anonymous reporting with identity separation
- 🗺️ **Live Task Map** — Leaflet.js geolocation lets officials assign workers to precise complaint coordinates

**Architecture:**
```
Citizen / Official / Worker Browser
    ↓
Spring MVC DispatcherServlet
    ↓
RoleFilter → Karma Interceptor → Performance Logger
    ↓
Controller Layer (5 role-scoped controllers)
    ↓
Service Layer (NLP classifier, Haversine engine, karma ledger)
    ↓
DAO Layer (Hibernate + Criteria API spatial queries)
    ↓
MySQL 8.0 — spatial indexes, karma ledger, audit trail
```

</details>

<details>

**Features:**

- 📊 **Farmer Reputation Score** — Performance index using delivery history, quality ratings, and distance factors

## 🏗️ Architecture & Design Principles

<details>
<summary><b>Layered MVC Architecture (click to expand)</b></summary>

```
Browser / REST Client
        │
        ▼
 DispatcherServlet              ← Spring MVC front controller
        │
        ▼
   @Controller / @RestController ← Request handling, validation, routing
        │
        ▼
   @Service                     ← Business logic, @Transactional boundaries
        │
        ▼
   @Repository / DAO             ← Hibernate ORM, Criteria API, JPQL
        │
        ▼
   MySQL / PostgreSQL            ← Relational persistence, indexes, procedures
```

</details>

<details>
<summary><b>Design Patterns Used</b></summary>

| Pattern | Where Applied |
|---------|--------------|
| DAO Pattern | `GenericDaoImpl` base + entity-specific implementations |
| Service Layer | Transactional business logic, isolated from controllers |
| Strategy | Slot allocation algorithm variants by priority type |
| Observer | Notification triggers on order, advisory, karma events |
| Repository | Reusable query methods with Criteria API |
| Decorator | Optional prescription draft saving before finalization |

</details>

<details>
<summary><b>Security Checklist</b></summary>

- ✅ BCrypt password hashing (never plaintext storage)
- ✅ CSRF tokens on all state-mutating operations
- ✅ Role-Based Access Control (ADMIN · DOCTOR · PHARMACIST · USER)
- ✅ JWT for stateless REST API authentication
- ✅ AES encryption for anonymous/whistleblower data
- ✅ Path traversal protection on multipart file uploads
- ✅ SQL injection prevention via Hibernate + parameterized queries

</details>
